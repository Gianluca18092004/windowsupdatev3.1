如何解除安裝軟體修正檔？
有些修正檔可使用 Windows Update 網站上的 [產品更新] 網頁，來解除安裝：
  1. 移至 [產品更新] 網頁。
  2. 如果出現提示訊息，請按一下 [是]。
  3. 出現 [產品更新] 網頁時，請按一下位於網頁上方的 [顯示已更新組件]。
  4. 如果可以解除安裝修正檔，則修正檔說明的右方會出現 [解除安裝] 按鈕。
  5. 按一下您想解除安裝的修正檔說明旁的 [解除安裝]。
無法由 [產品更新] 解除安裝的修正檔，可能也無法以其他方法解除安裝。請參閱每個修正檔的 [請先閱讀] 頁，以取得額外的解除安裝資訊。

裝置驅動程式支援
免費的安裝/解除安裝可以透過電子郵件在 Windows Update 網站支援上取得。有關驅動程式的安裝/解除安裝以外的問題，可從 http://support.microsoft.com/directory 找到完整的支援選項清單。在某些情況下，支援費用會不同，請查看網站，以取得詳細資訊。

常見問題
什麼是 Windows Update？
Windows Update 是 Windows 的線上延伸，其設計目的是協助您將電腦保持在最新狀態。Windows Update 有一個中央類別，稱為產品更新，它包括新的產品增強元件，例如系統檔案、裝置驅動程式、Service Pack 和新的 Windows 功能。建議您針對特定電腦所需，來使用這些增強元件。

什麼是產品更新？
[產品更新] 網頁是一個元件目錄，放置 Windows 修正檔和很多在 Windows 上執行程式的修正檔。選擇您想要的元件，以及從網頁下載和安裝到您的電腦。

為什麼我收到有關「此網頁 ActiveX 控制項不安全」的訊息？
當您將 Internet Explorer 的安全性設定值設為高時將出現此訊息。若要讓 Windows Update 網站可以順利作業，您必須將瀏覽器變更為中或較低的安全性。若要如此，請按一下 [工具] 功能表，然後按一下 [網際網路選項]。選取 [安全性] 索引標籤，然後按一下 [網際網路]。

什麼是 ActiveX 控制項？
ActiveX 控制項是結合 ActiveX 技術的可重覆使用軟體元件，這些元件可以用來新增如動畫或蹦現式功能表等特殊功能到網頁、桌面應用程式、軟體發展工具。Windows Update 使用 ActiveX 控制項來檢視您的電腦目前安裝了什麼軟體，以便提供您修正檔案以及其他可供您下載的軟體的正確清單。

瀏覽 Windows Update 網站的最佳解析度是多少？
Windows Update 網站上的網頁是以 800 x 600 或更高的螢幕解析度來檢視。若要設定解析度，請按一下 [開始] 按鈕，指向 [設定]，然後按一下 [控制台]。再按兩下 [顯示器] 圖示，然後按一下 [設定] 索引標籤。

有關我電腦的資訊會傳送到 Microsoft 嗎？
不。每次當您到我們的產品更新目錄時，Windows Update 會使用控制項檢查是否有適合您電腦的修正檔。此檢查是在您電腦上執行，以確保您資料的隱私權與安全性，且不會傳送任何資訊到 Microsoft。

如何將我的意見寄回 Windows Update 團隊？
您可以按一下每個網頁上方的意見信箱，然後按 [Windows Update]，將您對此網站的意見寄回 Windows Update 團隊。

如果 Microsoft TechNet Security 網站上有新的安全性快報， (這是中文網站)。 它也會出現在 Windows Update 上嗎？
安全性快報在 TechNet Security 站台上公佈的時間，和安全性修正檔在 Windows Update 上出現的時間，通常會有些差距。這是因為 Windows Update 會定期增加新的內容。雖然 Windows Update 致力於迅速發佈重要的安全性修正檔，但是新的內容也必須經過測試及驗證，以確保 Windows Update 為您提供的一定是您特定電腦所需的修正檔。通常，受支援產品的修正檔出現在 Windows Update 上的時間，會比它們在 TechNet 站台上發佈的時間晚幾天到幾個星期。如果您知道哪個特定版本的修正檔適用於您的電腦，又不想等待 Windows Update，則可以直接從安全性快報網頁下載修正檔。

Windows Update 支援哪些作業系統及程式？
Windows Update 提供的安全性修正檔可適用於 Windows 98、Windows 98 Second Edition、Windows Millennium Edition (Windows Me)、Windows NT 4.0、Windows 2000、Internet Information Server 4.0 (IIS 4.0)、Internet Information Services 5.0 (IIS 5.0) 及 Internet Explorer。然而，您只會看到適用於您特殊電腦的修正檔。Windows Update 不支援某些伺服器作業系統及伺服器型的程式。

不支援哪些伺服器產品呢？
Windows Update 起初是一個為 Windows 用戶的家用電腦提供修正檔的網站。後來 Windows Update 逐漸擴充，提供的修正檔也涵蓋了一些家庭常用的伺服器產品 (像是 IIS)。然而，Windows Update 無法支援特定用於某些伺服器產品 (如 Windows 網域控制站、Microsoft Exchange Server、Microsoft SQL Server、Microsoft ISA Server 等) 的安全性修正檔。使用這些產品的用戶請查閱 Microsoft TechNet Security 網站上的安全性快報，以取得有關下載這些產品的修正檔之位置的相關資訊。 (這是英文網站)。

Windows Update 是否持續提供 Internet Information Server 4.0 (IIS 4.0) 及 Internet Information Services 5.0 (IIS 5.0) 的安全性修正檔和其他修正檔？
否。Windows Update 在 2001 年 5 月 23 日首次提供一整套 IIS 5.0 的重大修正檔。此套件並未包括當時已發佈的所有 IIS 修正檔，但它提供了從 2000 年 3 月到目前為止的最新修正檔的組合。如需 2000 年 3 月之前所發佈的修正檔，請到 TechNet page for Windows Web services (IIS)。 (這是英文網站)。 如果建立了 IIS 的新安全性修正檔或其他重大修正檔，則此套件會由一組包括新的修正檔及先前套件之所有修正檔的套件所取代。這樣的話，無論您多久造訪一次 Windows Update，都可以取得 IIS 產品的最新修正檔。

如何取得與安全性相關的所有修正檔的資訊？
您可以在 Microsoft TechNet Security 網站檢視所有的安全性快報。您也可以訂閱 Microsoft Security Notification Service 並收到有關最近發佈的安全性快報的電子郵件警示。 (這些是英文網站。)
