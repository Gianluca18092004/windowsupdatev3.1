サポート情報
 Windows Update Web サイトに問題がある場合は、オンライン サポートをご利用いただけます。次のオプションのいずれかを選択してください。

次のサポート オプションを選択する際に、Microsoft "オンライン プロファイル" の作成が要求される場合があります。オンライン プロファイルとは、Microsoft からインターネットでのサポートを受けるためのチケットにあたるものです。


ソフトウェアのアップデートに対するサポート : Windows Update で提供されているソフトウェアのアップデートに関するサポート情報を参照できます。

デバイス ドライバに対するサポート : Windows Update で提供されているデバイス ドライバのサポート情報を参照できます。

よく寄せられる質問: Windows Update に関する質問がある場合は、まず質問への回答の一覧を確認してください。

確認されている問題: Windows Update サイトに現在用意されている、「確認されている問題」の一覧を参照することもできます。

フィードバック: クイック調査のフォームを使ってコメントと提案をご記入ください。


Windows Update について

このアイコンは、このサイトの全ページで最上部に表示されます。アイコンをクリックすると、Windows Update サイトの最適な使用方法に関する説明が表示されます。

ソフトウェアのアップデートに対するサポート
ソフトウェアのアップデートに対するサポートについては、http://support.microsoft.com/directory/ で各種オンライン サポート サイトを参照してください。MSN 製品については、http://help.msn.co.jp/ を参照してください。サポートの内容によっては、料金が必要になる可能性があります。詳細については、ページの内容を確認してください。

日本以外の地域にお住まいの場合は、お住まいの地域のサポート情報を参照してください。 最新の情報については、サポート技術情報のサイト を表示して、[Worldwide サポート情報] の中から地域を選択してください。お住まいの地域に Microsoft の支社がない場合は、Microsoft 製品を購入した販売元に連絡してください。

弊社によるサポート サービスの価格、およびサービスを受けるための条件は、予告なしに変更する可能性があります。

ソフトウェアのアップデートをアンインストールするには
一部のアップデートは、Windows Update サイトの [製品の更新] ページを使ってアンインストールできます。
  1. [製品の更新] ページに移動します。
  2. メッセージが表示される場合は、[はい] をクリックします。
  3. [製品の更新] ページが表示されたら、最上部の [すべて表示] ボタンをクリックします。
  4. アップデートをアンインストールできる場合は、 アップデートの説明の横に [アンインストール] ボタンが表示されます。
  5. アンインストールするアップデートの説明の横に表示されている [アンインストール] をクリックします。
[製品の更新] からアンインストールできないアップデートは、ほかの方法を使ってアンインストールできる可能性があります。アンインストールに関する補足情報については、各アップデートの [最初にお読みください] ページを参照してください。

デバイス ドライバのサポート
ドライバに対するサポートについては、http://support.microsoft.com/directory でサポート オプションの一覧を参照してください。サポートの内容によっては、料金が必要になる可能性があります。詳細については、ページの内容を確認してください。

日本以外の地域にお住まいの場合は、お住まいの地域のサポート情報を参照してください。最新の情報については、サポート技術情報のサイト を表示して、[Worldwide サポート情報] を選択してください。お住まいの地域に Microsoft の支社がない場合は、Microsoft 製品を購入した販売元に連絡してください。

弊社によるサポート サービスの価格、およびサービスを受けるための条件は、予告なしに変更する可能性があります。

デバイス ドライバをアンインストールするには
重要 Windows Update からダウンロードしたドライバのアップデートをアンインストールするには、デバイスに付属するオリジナルのドライバを再インストールできることが必要です。ドライバまたはドライバのアップデートを、オリジナルのドライバに置き換えずに削除すると、ハードウェア デバイスが停止します。

デバイス マネージャを使って、お使いのコンピュータからドライバのアップデートを削除できます。

  1. デスクトップの [マイ コンピュータ] アイコンを右クリックします。
  2. [プロパティ] をクリックし、[デバイス マネージャ] タブをクリックします。
  3. ハードウェア デバイスを一度だけクリックして選択し、[プロパティ] をクリックします。
  4. [ドライバ] タブをクリックし、[ドライバの更新] をクリックします。
  5. [デバイス ドライバのアップグレード] ウィザードが開始します。[ドライバの場所を指定する (詳しい知識のある方向け)] を選択して、[次へ] をクリックします。
  6. [このデバイスで認識済みのドライバの一覧を表示し、使用するドライバを選択する] を選択して、[次へ] をクリックします。
  7. [ディスク使用] をクリックし、[参照] をクリックして、インストールするドライバ用の .inf ファイルの場所を指定します。この操作によって、Windows Update から入手したドライバは削除され、ハードウェア デバイスのオリジナルのドライバに置き換わります。
注意 Windows Update に戻ると、アンインストールしたのと同じドライバが利用可能と表示される可能性があります。このドライバは、インストールしないでください。Windows Update から入手したドライバのアップデートをアンインストールする手順について、質問または問題がある場合は、サポート情報のリンクを参照してサポートまでご連絡ください。

よく寄せられる質問
Windows Update とはどのようなものですか ?
Windows Update は Windows のオンライン版拡張機能で、お使いのコンピュータを常に最新の状態にするように設計されています。Windows Update には、"製品の更新" と呼ばれるカタログがあります。このカタログには、システム ファイル、デバイス ドライバ、サービス パック、Windows の新機能などの新しい製品拡張機能が含まれています。ここでは、これらの拡張機能の中で、お使いのコンピュータに必要なもののダウンロードをお勧めしています。なお、ハードウェア製造元によっては、お使いの環境により適したデバイス用のモジュールを提供している場合があります。事前にハードウェア製造元にご確認ください。

[製品の更新] とはどのようなものですか?
[製品の更新] ページは、Windows および Windows 上で動作する多くのプログラムに対する修正プログラム、アップデート、および追加コンポーネントのカタログになっています。カタログを見て、好きなだけコンポーネントを選択し、そのコンポーネントを Web からコンピュータに直接ダウンロードおよびインストールすることができます。

このページの ActiveX コントロールは安全ではないというメッセージが表示されるのはどうしてですか ?
Internet Explorer のセキュリティ レベルが [高] に設定されていると、このメッセージが表示されます。Windows Update サイトが適切に動作するには、ブラウザのセキュリティ レベルを [中] または [低] に設定する必要があります。セキュリティ レベルを設定するには、[ツール] メニューの [インターネット オプション] をクリックし、次に [セキュリティ] タブの [インターネット] をクリックします。

ActiveX コントロールとはどのようなものですか?
ActiveX コントロールは、ActiveX 技術を取り入れた再利用可能なソフトウェア コンポーネントです。このコンポーネントは、アニメーションやポップアップ メニューなどの特殊な機能を Web サイト、デスクトップ アプリケーション、ソフトウェア開発ツールに追加するために使用されます。Windows Update では、ダウンロードできるアップデートおよびほかのソフトウェアの正確な一覧を表示するために、ActiveX コントロールを使用して、コンピュータにインストールされているソフトウェアを調べます。

Windows Update サイトを表示するのに最適な解像度はどれくらいですか?
Windows Update サイトの Web ページは、800 x 600 ピクセル以上で表示するように設計されています。解像度を設定するには、[スタート] メニューの [設定] をポイントし、次に [コントロール パネル] をクリックします。[画面] アイコンをダブルクリックし、[設定] タブをクリックします。

コンピュータ内の情報が Microsoft に送信されるのですか?
いいえ。[製品の更新] カタログを表示するとき、Windows Update はお使いのコンピュータに必要なアップデートをチェックします。このチェックは、コンピュータ上でローカルに行われ、プライバシーとデータの安全を保証しています。 Microsoft へは、何の情報も送信されません。

Windows Update チームにフィードバックを送信するにはどうすればよいですか?
このサイトに関するフィードバックを Windows Update チームにお送りいただく場合は、各ページの一番上にある黒いツール バーの [フィードバック] をクリックし、[Windows Update] をクリックします。なお、お問い合わせについて、個別にお答えはしておりませんのでご了承下さい。

TechNet セキュリティ センター   に掲載されている修正プログラムは、Windows Update からも利用できるのでしょうか?
セキュリティ情報のページに修正プログラムが掲載されてから、その修正プログラムを Windows Update で利用できるようになるまでには、1 週間程度の期間をいただくことがよくあります。これは、Windows Update へのコンテンツの追加を、1 週間単位で行っているためです。Windows Update には、重要な修正プログラムをできるだけ早くリリースするように努めていますが、コンテンツを追加するには、それぞれのコンピュータに適した修正プログラムだけが Windows Update から間違いなく提供できることをテストする必要があります。通常は、サポート対象製品の修正プログラムが Windows Update に掲載されるまでに、セキュリティ情報のページでアナウンスされてから数日、または数週間かかります。お使いのコンピュータに使用できる修正プログラムのバージョンがわかっていて、Windows Update から入手できるようになるのを待てない場合は、セキュリティ情報の Web ページから修正プログラムを直接ダウンロードしてください。

Windows Update は、どのオペレーティング システムやプログラムをサポートしていますか?
Windows Update は、Windows 98、Windows 98 Second Edition、Windows Millennium Edition (以下、Windows Me)、Windows NT 4.0、Windows 2000、Internet Information Server 4.0 (以下、IIS 4.0)、Internet Information Services 5.0 (以下、IIS 5.0)、および Internet Explorer に使用できる修正プログラムを提供しています。ただし、表示される修正プログラムは、使用中のコンピュータに適用できるものだけです。オペレーティング システムのサーバー製品、およびサーバー製品用のプログラムは、Windows Update ではサポートしていません。

サーバー製品にはサポートしていないものがあるとのことですが、どの製品ですか?
Windows Update は、ホーム コンピュータ用 Windows のユーザーを対象に、修正プログラムを提供するのが目的で始まりました。しかし時間の経過とともに、家庭でも使用されることの多い一部のサーバー製品 (IIS など) の修正プログラムも、Windows Update から提供するようになりました。Windows ドメイン コントローラ、Microsoft Exchange Server、Microsoft SQL Server、Microsoft ISA Server など、特定のサーバー製品に対する修正プログラムは、Windows Update では提供していません。これらの製品をご利用の方は、TechNet セキュリティ センターに掲載されているセキュリティ情報を参照して、各製品用の修正プログラムをダウンロードしてください。  

Windows Update では、Internet Information Server 4.0 (以下、IIS 4.0)、および Internet Information Services 5.0 (以下、IIS 5.0) 用の修正プログラムやその他のアップデートをすべて提供してきたのでしょうか?
いいえ。Windows Update では、2001 年 5 月 23 日に初めて、IIS 5.0 用の修正プログラムの累積的パッケージを提供しました。このパッケージには、2000 年 3 月からパッケージのリリース日までの修正プログラムが含まれていますが、IIS 用のすべての修正プログラムが含まれているわけではありません。2000 年 3 月以前にリリースされた修正プログラムについては、TechNet Online - IIS を参照してください。   IIS 用の修正プログラムまたは重要な更新が作成されると、このパッケージは新しいパッケージに置き換えられます。新しいパッケージには、新しく追加された修正プログラムと、それまでのパッケージに含まれていたすべての修正プログラムが含まれています。そのため、Windows Update をどのくらいの間隔で利用していても、Windows Update を利用した時点での最新の IIS 用修正プログラムを、いつでも入手できます。

すべてのセキュリティ問題の修正プログラムに関する情報はどこで入手できますか?
すべてのセキュリティ情報を参照するには、TechNet セキュリティ センターをご利用ください。また、マイクロソフト プロダクト セキュリティ警告サービスに登録すると、最近リリースされたセキュリティに関する警告が電子メールで通知されます。
